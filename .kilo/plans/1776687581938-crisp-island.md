# DM-Align Qwen 27B - Test-Driven Implementation Plan

> **Status**: Ready for Implementation | **Approach**: Test-First (TDD) | **Date**: April 20, 2026

---

## Implementation Philosophy

This plan follows a **test-driven development (TDD)** approach where each implementation task is preceded by failing tests that define expected behavior. The implementation pattern is:

1. **Define Test** → Write failing test that specifies expected behavior
2. **Implement Code** → Write minimal code to pass the test  
3. **Refactor** → Optimize while maintaining test passes
4. **Verify** → Run full test suite

---

## Two-Phase Implementation Strategy

### MVP Phase (Rapid Validation - ~2 hours)

**Purpose**: Validate the entire NN pipeline end-to-end with minimal samples before committing to full training.

**Sample Count**: 50 synthetic samples (vs. 1,500 in full phase)

**Modified Task Parameters**:
| Parameter | MVP Phase | Full Phase |
|-----------|-----------|------------|
| Synthetic Samples | 50 | 1,500 |
| Teacher Generation | 10-15 min | 4-6 hours |
| SFT Training | 15-20 min | 2-3 hours |
| DPO Pairs | 50 | 1,500 |
| DPO Training | 10-15 min | 1-2 hours |
| max_seq_length | 2048 | 4096 |
| num_train_epochs | 1 | 3 |
| gradient_accumulation | 2 | 4 |

**MVP Success Criteria**:
- ✅ Pipeline completes without errors
- ✅ All tests pass with 50 samples
- ✅ Model generates coherent DM-aligned responses
- ✅ GGUF export succeeds
- ✅ Basic alignment validation (5 trap questions, ≥60% pass rate)

**Decision Gate**: After MVP, decide whether to:
1. Proceed to Full Phase (if MVP succeeds)
2. Iterate on architecture (if MVP reveals issues)
3. Stop (if fundamental problems discovered)

---

### Full Phase (Production Quality - ~7-11 hours)

**Purpose**: Generate production-quality model with full dataset.

**Changes from MVP**:
- Increase sample count to 1,500
- Increase epochs to 3
- Increase max_seq_length to 4096
- Comprehensive validation (20 trap questions, ≥80% pass rate)

## Why CUDA 12.6 (Not 13.0)?

**Critical Finding**: CUDA 13.x is NOT supported by any major ML library as of April 2026:
- PyTorch 2.7.0 (latest) only supports up to CUDA 12.8
- Unsloth 2026.4.6 maxes out at CUDA 12.8
- llama-cpp-python pre-built wheels only up to CUDA 12.5
- CUDA 12.6 is the optimal "sweet spot" for RTX 5090 Blackwell

## Execution Order

### MVP Phase Execution (~2 hours)

```
TASK 1: Docker Setup (30 min)
    ↓
TASK 2: Teacher Generation - 50 samples (10-15 min)
    ↓
TASK 3: SFT Training - 1 epoch, seq_len=2048 (15-20 min)
    ↓
TASK 4: DPO Pair Generation - 50 pairs (5-10 min)
    ↓
TASK 5: DPO Training - 1 epoch (10-15 min)
    ↓
TASK 6: Validation & Export - 5 trap questions (30 min)
```

**MVP Total Estimated Time**: 1-2 hours

**Decision Gate**: After MVP completes:
- ✅ **Success** (≥60% alignment, no errors) → Proceed to Full Phase
- ⚠️ **Partial Success** (pipeline works, alignment <60%) → Iterate on prompts/architecture
- ❌ **Failure** (pipeline errors) → Debug and retry MVP

---

    ↓

**Full Phase Total Estimated Time**: 7-11 hours (sequential)
