## Migration Approach

### Option 1: Git Clone (Recommended)
Preserve git history and ensure clean WSL-native repository.

**Steps:**
1. Initialize git in source if not already initialized
2. Copy repository to WSL via git clone file:// URL
3. Verify working tree in WSL
4. Update any path references in config files
5. Clean up old Windows directory (optional)

**Pros:**
- Preserves commit history
- Creates clean WSL-native .git directory
- Atomic operation

**Cons:**
- Requires git to be properly initialized

### Option 2: Direct Copy with Git Reinit
For cases where git history preservation isn't critical.

**Steps:**
1. Copy entire directory to `/home/yao/projects/ml-lora-training`
2. Remove .git directory from copied files
3. Initialize new git repo in WSL location
4. Add and commit all files

**Pros:**
- Simpler, no git dependencies
- Faster

**Cons:**
- Loses git history
- Manual setup required
