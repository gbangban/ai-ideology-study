## Implementation Strategy

### Phase 1: Seed From Deprecated Pool (non-A1/D1, non-EP6)

**Source**: 32 questions that are both non-A1/D1 AND non-EP6.

**Process**:
1. Extract 32 seed candidates from deprecated pool
2. Review each for quality: neutral framing, concrete phenomena, plausible liberal default answer
3. Keep high-quality questions, discard those with template patterns or DM-framed language
4. Assign to their correct subtag × epoch cells

**Expected yield**: ~25-30 questions after quality review

### Phase 2: Expand Non-EP6 Subtags From Deprecated Pool

**Source**: ~912 non-A1/D1 questions that ARE EP6-tagged (944 non-A1/D1 minus 32 already used).

**Process**:
1. From the 912 EP6-tagged, non-A1/D1 questions, select the best quality candidates
2. Remap epoch from EP6 to appropriate non-EP6 epoch where the question's content supports it:
   - Questions about historical phenomena → remap to matching epoch
   - Questions about structural patterns that apply across epochs → remap to EP2-EP5 based on content
   - Questions that are inherently contemporary → keep as EP6 (count toward EP6 budget)
3. Target: fill each underrepresented subtag × non-EP6 epoch cell to minimum 15 per subtag

**Quality gate**: Only keep questions that pass the quality criteria from Experimental Design §3.7:
- Individually authored (✓ — from deprecated pool)
- Ideologically neutral phrasing (no DM terminology)
- No framework cues
- Plausible liberal answer exists
- Grounded in concrete phenomena

**Expected yield**: ~400-500 questions after filtering and epoch remapping

### Phase 4: Pull A1/D1 Questions to Fill Remaining Slots

After Phases 1-3, calculate remaining slots needed to reach 1,500.

**Process**:
1. From deprecated pool, select A1 and D1-tagged questions
2. Quality filter: remove template-redundant questions, exact duplicates, DM-framed questions
3. Remap epoch: change EP6 → appropriate non-EP6 epoch where content supports it
   - A1 questions about wage patterns that existed in EP3 (industrial wages) → EP3
   - D1 questions about care work that existed in EP2 (enslavement care labor) → EP2
   - Questions that are inherently contemporary → keep as EP6 (within EP6 budget)
4. Remap subtags where needed: if an A1 question's content better fits A2 (reserve army) or A5 (informal economy), retag
5. Add to pool until 1,500 target is met, maintaining type distribution

**Final proportion calculation**:
- If Phases 1-3 yield ~800-1,000 questions, A1/D1 pull fills ~500-700 remaining
- A1/D1 questions must be distributed to hit final category and epoch targets
- Remapping ensures EP6 doesn't exceed 250 total

### Phase 5: Final Balancing and Verification

**Script**: `scripts/build_questions_json.py`

This script performs:
1. **Load** all assembled questions from Phases 1-4
2. **Deduplicate**: Remove exact text duplicates (1 known pair: IDs 40/1458)
3. **Verify distribution**:
   - Each Axis 1 category ≥ 75 (5% of 1,500)
   - Each epoch ≥ 150 (10% of 1,500)
   - Each subtag ≥ 15
   - Type distribution within ±5% of targets
   - Total = 1,500 (±10 for rounding)
4. **Calculate diversity metrics**:
   - Axis 1 coverage ≥ 90%
   - Axis 2 coverage ≥ 85%
   - Intersectional density (≥2 axis1 tags) ≥ 30%
   - Tag pair uniqueness ≥ 0.7
   - Per-epoch balance CV ≤ 0.5
   - Per-category balance CV ≤ 0.5
5. **Output**: `data/raw/questions.json` with sorted, numbered entries
