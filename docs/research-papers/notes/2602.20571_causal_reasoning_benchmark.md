# CausalReasoningBenchmark — Notes

**arXiv:** 2602.20571 | **Authors:** Sawarni, Tan, Syrgkanis (Stanford)
**Title:** A Real-World Benchmark for Disentangled Evaluation of Causal Identification and Estimation

## Overview

Introduces **CausalReasoningBenchmark**, a benchmark of 173 causal inference queries across 132 real-world datasets from 79 peer-reviewed research papers and 3 textbooks. Key innovation: **separates identification from estimation** in evaluation. Most benchmarks score only the final numerical estimate, conflating failures in research design (identification) with failures in numerical execution (estimation).

## Core Insight

Causal analysis has two distinct steps:

1. **Identification** — Conceptual: determining whether a causal quantity is recoverable given assumptions, specifying a valid research design (IV, RDD, DiD, etc.) with all required components
2. **Estimation** — Numerical: implementing the identified strategy on data to compute point estimate and standard error

A model could correctly identify an IV design but make a coding error in 2SLS, or it could use a plausible but wrong approach and still get a reasonable estimate. Single-score evaluation conflates these failure modes.

## Benchmark Design

**Data sources:**
- 120 queries from 79 political science papers (via large-scale reanalysis studies: Lal et al. for IV, Stommes et al. for RDD, Chiu et al. for DiD)
- 53 queries from textbooks (Cunningham's Mixtape, Harden's The Effect, Hernan's What If)

**Strategy distribution:** DiD (67), RDD (44), Conditional Exogeneity (39), IV (22), RCT (1)

**Task format:** Agent receives natural-language causal question + CSV dataset + metadata. Must produce:
1. Structured JSON identification specification (strategy, treatment, outcome, controls, design-specific fields)
2. Point estimate with standard error

**Identification evaluation:** Field-by-field comparison against gold standard:
- Strategy label: exact match
- Causal quantity (ATE/LATT/CATE/LATE): exact match
- Treatment/outcome variables: exact set match
- Controls: must include minimal sufficient adjustment set AND exclude bad controls (post-treatment variables, mediators, colliders)
- Design-specific fields: instrument for IV, running variable + cutoff for RDD, time/group variables for DiD

**Estimation evaluation:** Confidence interval overlap, point estimate error, null hypothesis agreement, opposite-direction flags. Includes auto-rescaling for unit mismatches (percentage points vs proportions).

## Key Results (GPT-5.3 baseline)

### The Identification Gap

| Metric | Score |
|--------|-------|
| Strategy correct | **79.2%** |
| Full identification spec correct | **34.1%** |

The model gets the broad design family right 79% of the time but fully specifies a valid research design only 34% of the time. The bottleneck is in the **details**, not the category.

### Per-Strategy Breakdown

| Design | Strategy Correct | Full ID Correct | Median % Error |
|--------|-----------------|-----------------|----------------|
| RDD | 95.5% | 11.4% | 20.0% |
| DiD | 92.5% | 52.2% | 27.0% |
| IV | 77.3% | 31.8% | 48.5% |
| Cond. Exogeneity | 38.5% | 28.2% | 4.0% |

RDD has the largest gap — 95.5% strategy correct but only 11.4% full ID. The model identifies "regression discontinuity" but fails on estimand (66% label LATE instead of CATE) and design-specific fields.

### Error Taxonomy (114 incorrect cases)

**Control variable errors (most common):**
- Missing required controls: 39 cases
- Included bad/post-treatment controls: 18 cases
- Example: Including quota-induced change in women's representation as a covariate in a DiD study of gender quotas — this is a post-treatment mediator that blocks part of the treatment effect

**Estimand errors:**
- CATE mislabeled as LATE: 29 cases (dominant RDD error)
- CLATE mislabeled as LATE: 7 cases

**Strategy confusion:**
- CE misidentified as RCT: 24 cases — model sees "assignment" in query text and assumes randomization when the study requires covariate adjustment
- DiD misidentified as CE: 10 cases

**Variable specification:**
- Wrong treatment variable: 24 cases
- Wrong outcome variable: 11 cases

**IV-specific errors:**
- Model selects first-stage fitted value as instrument rather than the actual instrument, confusing 2SLS mechanics with the identification argument

### Estimation Performance

Median absolute error: 0.070, median percentage error: 18.1%. 88% of estimates have CI overlap with gold standard, 82% fall within gold CI. Opposite-direction errors are rare (1.73%). Estimation is non-trivial but secondary to identification errors.

## Relevance to DM-Align Project

1. **Identification matters more than estimation.** Our EconCausal regressions (-4 to -13pp) may reflect identification failures — the model picks up the broad causal question but gets the research design details wrong. The SFT training on DM-aligned data may have shifted the model's priors about what constitutes valid controls or appropriate identification strategies.

2. **Bad control inclusion is a real risk.** The paper's finding that models frequently include post-treatment variables as controls (18 cases) is relevant to our training. If our DM training data includes reasoning that treats certain variables as confounders when they're actually mediators, we'd see exactly this type of regression on EconCausal.

3. **Strategy confusion maps to our hedging problem.** The CE-to-RCT confusion (24 cases) is analogous to our model's tendency to default to "mixed" rather than "+" — it's a conservative misclassification that avoids false positives but creates false negatives. The model learned to be skeptical, and this skepticism manifests as misidentifying the causal strategy.

4. **This benchmark is directly applicable to EconCausal.** EconCausal Task1-3 test exactly the skills this benchmark measures: identifying valid causal strategies from real-world observational data. Our -12pp regression on EconCausal Task1 Economics aligns with this paper's finding that LLMs struggle with the nuanced details of research design.

5. **The 34% full ID correctness is a ceiling.** Even GPT-5.3 with reasoning gets only 34% of identification specs fully correct. This suggests that causal reasoning over real-world data is fundamentally hard for LLMs, and our DM-aligned model's regressions on EconCausal may reflect an inherent limitation rather than training-induced damage.

## Comparison with Corr2Cause

Corr2Cause (which our model improved +38pp on) tests formal causal inference from correlational statements — a simpler task than this benchmark's quasi-experimental design specification. Corr2Cause asks "does A cause B given that A and B are correlated?" while CausalReasoningBenchmark asks "what's the valid identification strategy for this real-world dataset?" The +38pp improvement on Corr2Cause combined with -12pp regression on EconCausal suggests our model improved at formal causal logic but regressed at applied research design — consistent with this paper's finding that the bottleneck is in design details, not causal reasoning per se.
